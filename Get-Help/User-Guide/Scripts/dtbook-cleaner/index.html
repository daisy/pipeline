DTBook Cleaner

The “DTBook cleaner” script applies fixing routines to a DTBook XML file. Three sets of routines are provided and can be individually activated : a “repair” set, a “tidy” set and a “narrator” set. Optionally, the script can apply sentence detection on the DTBook file.

More information on the fixes applied by the script for each set of routines is provided on the Detailed cleaning routines page.

Synopsis

Inputs

  • DTBook file(s)
    One or more DTBook files to be cleaned
    Media-type:
    • application/x-dtbook+xml
    Sequence: yes

Outputs

  • DTBook file(s)
    Cleaned DTBooks

Options

  • Apply sentences detection
    Encapsulate sentences within the document
    Possible values: true or false
    Default value: false
  • Include doctype in resulting DTBook(s)
    Include doctype in resulting DTBook(s)
    Include doctype in resulting DTBook(s)

    The resulting DTBook will have a standard DTBook 2005-3 doctype, optionally with MathML declaration if MathML is present in the document.

    Possible values: true or false
    Default value: true
  • Tidy - Externalize whitespaces
    Externalize leading and trailing whitespace
    Externalize leading and trailing whitespace

    from em, strong, sub, sup, pagenum, noteref.

    Possible values: true or false
    Default value: false
  • Cleanup the document for audio synthesis
    Apply cleaning routines on the document to prepare it for text-to-speech processes.
    Possible values: true or false
    Default value: false
  • Tidy - Document language
    Set a document language
    Possible values: Any <string>
    Default value:
  • Repair the DTBook
    Apply repair routines on the DTBook.
    Possible values: true or false
    Default value: false
  • Tidy up the DTBook
    Apply tidying routines on the DTBook.
    Possible values: true or false
    Default value: false
  • Narrator - Publisher
    Publisher metadata (dc:Publisher) to be added if none is defined in the DTBook
    Possible values: Any <string>
    Default value:
  • Tidy - Simplify headings layout
    Simplify the level structure
    Simplify the level structure

    Redundant level structure is sometimes used to mimic the original layout, but can pose a problem in some circumstances. By selecting this option the script simplifies the level structure by removing redundant levels (subordinate levels will be moved upwards). Note that the headings of the affected levels will also change, which will alter the appearance of the layout.

    Possible values: true or false
    Default value: false 
Usage: dp2 [GLOBAL_OPTIONS] dtbook-cleaner [OPTIONS]

Options:
--source source One or more DTBook files to be cleaned
--ApplySentenceDetection [ApplySentenceDetection] Apply sentences detection (default: false)
--WithDoctype [WithDoctype] Include doctype in resulting DTBook(s) (default: true)
--externalizeWhitespace [externalizeWhitespace] Tidy - Externalize whitespaces (default: false)
--narrator [narrator] Cleanup the document for audio synthesis (default: false)
--documentLanguage [documentLanguage] Tidy - Document language (default: )
--repair [repair] Repair the DTBook (default: false)
--tidy [tidy] Tidy up the DTBook (default: false)
--publisher [publisher] Narrator - Publisher (default: )
--simplifyHeadingLayout [simplifyHeadingLayout] Tidy - Simplify headings layout (default: false)
-o,--output [OUTPUT] Path where to store the results. This option is mandatory when the job is not executed in the background
-z,--zip Write the output to a zip file rather than to a folder
-n,--nicename [NICENAME] Set job's nice name
-r,--priority [PRIORITY] Set job's priority (high|medium|low)
-q,--quiet Do not print the job's messages
-p,--persistent Delete the job after it is executed
-b,--background Sends the job and exits
